Beta Test Information

This version of the SAM Shade Calculator is a Beta version for testing. During the Beta test period, we are collecting feedback to help us identify bugs and ways to improve the software.

To provide feedback, we prefer that you use the Shade Calculator Beta Test page on the SAM website, but you can also email us your feedback.

Known issues

The following is a list of issues we know about:

•Any intersecting objects in the scene causes the shade algorithm to miscalculate shade fractions.
•Box with no sides is invisible in bird's eye view.
•"A" shortcut (show all objects) does not work.
•Clicking Save opens a file browser window with a Save button instead of saving the file.
•Closing the calculator with a modified scene does not prompt to save changes.
•There is no option to clear the scene, or to create a new file without restarting the calculator.
•The import buttons on the Analyze page do not serve a purpose.

Wish list

The following items are things we would like to add to the calculator, but may not have the resources to do so in time for the release:

•Add option to rotate 3D scene view by typing solar azimuth and altitude values to set the view to specific sun position.
•In 3D scene view, show the time (month/day/hour) in addition to sun angles.
•The location workflow is awkward because it both sets the scene's location and provides tools for the image underlay. These tasks are both related to the scene's location, but the image underlay only affects bird's eye view. Consider moving underlay tools to bird's eye view.
•For triangular active surface, use base/altitude/rotation properties to define for more flexibility, for example upside down right triangle for an array on a dormer roof.
